# Student Mininet Rerun Guide

This branch is built as:

- S23 project code and artifacts from `moahaimen/s23_network`
- FIX1 Mininet/OVS live validation code and final hardfix evidence

Use this guide on the Linux Mininet VM, not native Windows.

## 1. Clone

```bash
git clone -b student-s23-mininet https://github.com/moahaimen/s_mininet.git
cd s_mininet
```

## 2. Install Python dependencies

```bash
python3 -m pip install --user --upgrade pip
python3 -m pip install --user -r requirements.txt
```

## 3. Check Mininet and OVS

```bash
sudo mn --test pingall
mn --version
ovs-vsctl --version
```

If Mininet is not installed:

```bash
sudo apt update
sudo apt install -y mininet openvswitch-switch iperf iproute2
```

## 4. Clean stale Mininet state

Run this before every fresh live rerun:

```bash
sudo mn -c
```

## 5. Run the fresh live SDN/Mininet validation

This runs Abilene and GEANT across the professor scenarios with 5 repetitions.

```bash
sudo env PYTHONPATH="$PWD:$HOME/.local/lib/python3.8/site-packages" \
  python3 scripts/phase1_5/run_sdn_mininet_clean.py \
  --mode mininet \
  --topologies abilene geant \
  --scenarios normal single_link_failure two_link_failure capacity_degradation_50 spike_x3 mixed_spike_failure \
  --runs 5 \
  --artifact-prefix student_live_sdn_rerun
```

Output files are written under:

```text
results/gnn_lpd_dqn_selective_db_lp/sdn_mininet_clean/
```

Expected output names:

```text
student_live_sdn_rerun_per_run.csv
student_live_sdn_rerun_summary.csv
student_live_sdn_rerun_validation.md
student_live_sdn_rerun_raw_logs/
```

## 6. Faster smoke run

Use this first if the VM is slow:

```bash
sudo mn -c
sudo env PYTHONPATH="$PWD:$HOME/.local/lib/python3.8/site-packages" \
  python3 scripts/phase1_5/run_sdn_mininet_clean.py \
  --mode mininet \
  --topologies abilene \
  --scenarios normal \
  --runs 1 \
  --artifact-prefix student_smoke_mininet
```

## Notes for reporting

- Do not reuse older CSVs as fresh evidence.
- Use only rows generated by the current live rerun.
- Keep startup time separate from install and recovery timing.
- Keep raw logs with the summary CSV when sending results.
